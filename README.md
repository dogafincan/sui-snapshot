# Sui Holders Snapshot

TanStack Start web app for running Sui holder snapshots on Cloudflare Workers.

The app:

- pages Sui GraphQL for every live `Coin<T>` object of a target token,
- aggregates balances by owner address,
- optionally computes proportional airdrop allocations with exclusions,
- renders the full result set in a TanStack Table UI,
- exports the same rows as CSV without rerunning the snapshot.

## What It Does

Input:

- Sui coin type in `0xPACKAGE::MODULE::TOKEN` format
- optional airdrop amount
- optional excluded addresses

Output:

- ranked holder table
- optional `airdrop_amount` column
- client-side CSV download

## Stack

- TanStack Start
- Cloudflare Workers via `@cloudflare/vite-plugin`
- shadcn/ui + Tailwind CSS v4
- TanStack Table
- Vitest

## Project Structure

- `src/routes/index.tsx`: route entrypoint for `/`
- `src/components/snapshot-workbench.tsx`: form, loading states, summary cards, download action
- `src/components/holders-table.tsx`: TanStack Table configuration, sorting, filtering, pagination
- `src/lib/sui-snapshot.server.ts`: Sui GraphQL execution and aggregation
- `src/lib/sui-snapshot.functions.ts`: TanStack Start server function wrapper
- `src/lib/sui-snapshot.ts`: shared validation, formatting, CSV, and allocation helpers
- `wrangler.jsonc`: Cloudflare Worker configuration

Generated files:

- `src/routeTree.gen.ts`
- `worker-configuration.d.ts`

These are generated by the framework/tooling and should not be treated as normal hand-edited source files.

## Run Locally

1. Install dependencies:

   ```bash
   npm install
   ```

2. Start the local dev server:

   ```bash
   npm run dev
   ```

3. Open [http://localhost:3000](http://localhost:3000).

Production-style local preview:

```bash
npm run preview -- --host 127.0.0.1
```

That serves the built app on `http://127.0.0.1:4173`.

The default Sui GraphQL endpoint is `https://graphql.mainnet.sui.io/graphql`.

### Optional endpoint override

If you want to point the app at a different GraphQL endpoint, provide
`SUI_GRAPHQL_ENDPOINT` through your Cloudflare Worker environment for local or
deployed runs. If the variable is absent, the app falls back to mainnet.

If you introduce more Worker variables later, add them to both `wrangler.jsonc`
and this README.

## Scripts

- `npm run dev` starts the local TanStack Start dev server on port `3000`.
- `npm run build` builds client and server bundles for Workers.
- `npm run preview` builds and serves the production bundle locally.
- `npm run test` runs the Vitest suite.
- `npm run deploy` builds and deploys with Wrangler.
- `npm run cf-typegen` regenerates `worker-configuration.d.ts` from `wrangler.jsonc`.

## Deployment

Deploy to Cloudflare Workers with:

```bash
npm run deploy
```

The Worker entrypoint is `@tanstack/react-start/server-entry`, configured in
`wrangler.jsonc`.

## Verification

Useful checks before deploying:

```bash
./node_modules/.bin/tsc --noEmit
npm test
npm run build
```

## Notes

- The app is stateless and public by design. No D1, KV, R2, or background jobs.
- Snapshot accuracy matches the original script: it reflects live state while the
  Worker pages through results, so balances can drift slightly during execution.
- For checkpoint-perfect accounting, use an indexer over Sui checkpoints instead of
  live GraphQL pagination.
- The original CLI script has been removed; this repository is now web-app-first.
